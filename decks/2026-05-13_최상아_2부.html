[전공과목 — 자기 학습의 깊이] · OS (A) — 커널을 분석하고 코드를 한 줄씩 읽으며 그 흐름을 보는 데서 재미를 느꼈습니다. 최고의 개발진이 만들었다고 생각하며 공부했던 기억이 납니다. · DB (C+) — 학점은 낮지만 제가 선택한 길이기 때문입니다. SQL을 직접 사용하며 몸으로 익혔고, DB 분석가가 되기 위한 가장 기초이기에 선택했습니다. · 빅데이터 (B+) — Hadoop, R로 DB 관리 방향을 잡고, DB 분석을 활용한 SW 개발자가 되기 위한 첫걸음이었습니다. [프로젝트 — Simple Merge (JAVA)] 두 개의 txt 파일을 중복 없이 비교·합치는 프로그램. Vector → JList 출력 과정에서 String 변환 문제 발생 → 차선책으로 한쪽은 JList, 다른 쪽은 String으로 열어 EditView 통해 수정 가능하도록 구현. [미래 — DBA를 넘어선 BigData 활용 SW 개발자] 단순 경영뿐 아니라 모든 분야에서 활용 가능한 기술 개발. 각 팀이 원하는 DB 분석을 쉽게 할 수 있는 SW 개발자가 되고 싶습니다.
합격 포인트 ①
약점 인정 + 비전 연결 — DB C+ 학점을 정직하게 인정하면서도 ‘DB 분석가가 되기 위한 기초’라는 명확한 직무 비전과 연결.
합격 포인트 ②
‘재미·동기’ 중심의 학습 서사 — OS 수업에서 커널 코드를 한 줄씩 읽으며 흐름을 분석한 경험. 단편적 활용이 아닌 깊이 있는 학습.
합격 포인트 ③
실제 프로젝트 + 트러블슈팅 디테일 — Simple Merge의 String 변환 문제 + 차선책 도출까지 구체적으로.
‘완벽한 스펙’이 아니라 ‘왜 이 직무를 원하고, 어떻게 학습했는지’가 합격을 만든다.
