PART 01 PRINCIPLES

디자인 원칙 4가지

01
컨설팅 톤, 교육 무게감
  • 프로모션이 아닌 ‘리포트’에 가까운 인상
  • 여백·정렬·표 중심, 장식은 최소화
  • 신뢰도가 곧 설득력
02
각진 모서리, 얇은 구분선
  • border-radius: 0
  • 1px 라인으로 구조를 분명히
  • 데이터를 시각적으로 정직하게
03
한 톤의 네이비
  • 주색 단 1가지: #1C2885
  • 파스텔 블루는 강조 블록에만
  • 빨강은 ‘리스크/주의’ 신호용 제한
04
Pretendard 단일 서체
  • 한글·영문·숫자 모두 Pretendard
  • 두께·크기로만 위계 형성
  • PPTX는 ea/cs 폰트 슬롯까지 지정

Template 12 — case_analysis

이력서·포트폴리오 분석 — 3 합격 포인트 + 결론

샘플 이력서 (요약)
개발자의 역할은 결국 문제를 기술로 해결하는 것이라고 생각합니다. [고객의 입장에 서] 일정 조율 서비스 ‘OneTime’ 직접 개발·운영, 누적 사용자 7,500명+. ‘내 스케줄’ 접근성 문제로 실사용자 5명에 그쳤으나, 에브리타임 시간표 연동 기능 구현 후 80명+ 증가. [레거시 기반 개선] SKT 기업 프로젝트에서 데보션 마이페이지 개선 — Java 8/JSP/MyBatis 레거시 분석, 신규 기능 추가, 컨벤션 준수.
합격 포인트 ①
일정 조율 서비스 직접 개발·운영하며 실사용 서비스 운영 경험을 보여주고 있음.
합격 포인트 ②
팀 프로젝트에서 반복 업무 효율성·정확도 개선을 이끌어냄.
합격 포인트 ③
SKT 기업 프로젝트 + Java/JS/JSP/MyBatis 등 실무 스택 활용 능력 보유.
실사용 서비스 운영 경험 + 효율 개선으로 실무 적용 가능 역량을 보여준 것이 합격의 핵심.
